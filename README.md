# Planeación_Proyecto_Emerson

Este repositorio tiene como propósito organizar y documentar la planeación del proyecto personal. Aquí defino las actividades iniciales, el orden de trabajo y la forma en que utilizaré GitHub para gestionar las tareas y el seguimiento.

## Objetivo del repositorio
Implementar una estructura clara de planeación usando Git y GitHub, aplicando:
- control de versiones,
- uso de ramas,
- commits descriptivos,
- issues para las tareas,
- y pull requests para integrar los cambios.

## Fases planificadas del proyecto

1. **Requisitos y análisis inicial**  
   Identificar las necesidades del proyecto, usuarios y funcionalidades principales.

2. **Diseño técnico**  
   Definir arquitectura, herramientas, estructura del repositorio y lineamientos técnicos.

3. **Planeación por actividades**  
   Elaborar un plan paso a paso usando issues, milestones y tablero Kanban.

4. **Implementación inicial (iteración 1)**  
   Crear la estructura básica del proyecto y los primeros módulos.

5. **Pruebas y validación**  
   Realizar pruebas básicas para asegurar funcionamiento correcto.

6. **Documentación y cierre**  
   Actualizar documentación final, conclusiones y entregables.

---

Este repositorio es parte de la evidencia de la Actividad Desescolarizada (Sesión 2), donde aplico lo aprendido sobre Git, GitHub y la organización del trabajo con control de versiones.
