## Preguntas reflexivas

### ¿Qué dificultades encontraste al usar GitHub para planificar tu proyecto?
Al principio me costó entender cómo se conectaba todo: las ramas, los commits y los pull requests. A veces no sabía si estaba trabajando en la rama correcta o si debía crear otra. También me confundía un poco dónde encontrar cada opción dentro de GitHub. Pero a medida que fui practicando, las cosas empezaron a tener más sentido y ya me sentí más cómodo con el proceso.

### ¿Qué beneficios te aportó GitHub para organizar las tareas y visualizar el avance?
Lo que más me ayudó es que GitHub me permite ver mi progreso paso a paso. Cada commit queda guardado y puedo regresar a ver cuándo hice algo o qué cambié. También me gustó que puedo tener todo separado por ramas para no desordenar el trabajo. En general, GitHub me ayudó a tener más claridad y orden en la planeación.

### ¿Cómo podrías integrar el control de versiones en un entorno laboral real?
En un trabajo real, usaría GitHub para que cada persona tenga su parte del proyecto sin dañar lo que ya existe. Se pueden revisar cambios antes de unirlos, trabajar en ramas por funciones y llevar una historia clara de lo que se hace. Eso evita confusiones y asegura que todos estemos trabajando en la misma dirección.

### ¿Qué aspectos del trabajo colaborativo aprendiste que podrías aplicar en un equipo profesional?
Aprendí que es importante dejar mensajes claros, porque otros dependen de lo que uno escribe. También entendí que las ramas ayudan a evitar problemas entre compañeros y que los pull requests sirven para revisar el trabajo antes de mezclarlo. Aunque trabajé solo, pude ver lo útil que sería esto en un equipo grande.
